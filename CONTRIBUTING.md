# Contributing to B#

First of all, thank you for considering contributing to B#.

B# is a human-centric programming language focused on readable, structured English-like syntax. Contributions help shape its future.

---

## How to Contribute

You can contribute in the following ways:

- Reporting bugs
- Suggesting language improvements
- Improving documentation
- Enhancing the interpreter
- Building tooling (LSP, formatter, debugger)
- Improving performance
- Writing tests

---

## Development Setup

1. Clone the repository
2. Install dependencies (if applicable)
3. Run the interpreter locally
4. Test changes using example `.bsharp` programs

---

## Branching Model

- `main` → stable releases
- `dev` → active development
- feature branches → `feature/your-feature-name`

Example:
feature/module-system
fix/parser-recursion

---

## Code Style Guidelines

- Keep syntax human-readable
- Avoid introducing unnecessary symbols
- Follow existing parser/runtime architecture
- Add comments for non-trivial logic
- Keep functions small and modular

---

## Commit Message Format

Use clear, structured commit messages:

feat: add dictionary key access
fix: resolve recursion stack bug
refactor: simplify expression parser
docs: update language specification

---

## Pull Request Process

1. Fork the repository
2. Create a feature branch
3. Write clean commits
4. Ensure examples still work
5. Open a pull request with a clear explanation

---

## Language Design Philosophy

B# prioritizes:

- Readability over brevity
- Explicit structure over implicit magic
- Human-like syntax over symbolic density
- Clear runtime behavior
- Educational clarity

---

## Questions?

Open an issue or discussion thread in the repository.

We welcome thoughtful contributions.
